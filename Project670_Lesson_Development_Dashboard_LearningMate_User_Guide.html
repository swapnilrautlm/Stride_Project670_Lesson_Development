Project 670 — Dashboard User Guide

A walk-through of every tab — Delivery Plan May 21, Live Delivery Plan, Action Required, Live Status & RAG, Insights, and the new Drill Down — plus the underlying Google Sheet model and worked examples of how to fill it in.

1 What this dashboard shows

The dashboard answers the question stakeholders ask most often: "When are we delivering what, and to whom?"

It focuses on the final delivery milestone — "Deliver and SO" (LearningMate) — for every course in Project 670, broken down by State, Subject, Grade, and Derivative type.

Every chart, table, and calendar reads from the same underlying course list (991 trackable courses across 4 subjects and 30 MPS states), so picking a filter at the top instantly recalculates everything below.

2 Opening the dashboard

  1. Open the dashboard by clicking the link your administrator shared. It loads directly in your browser (Chrome, Edge, or Safari recommended).
  2. Wait a moment for the page to load. The header reads Project 670 Dashboard · Lesson Development · Status & Risk.
  3. If a section looks empty when the page first loads, scroll up and click Reset in the filter bar — it clears any stale filter state from a previous session.
Tip: Force-refresh with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) any time the dashboard looks stuck.

3 The big picture

The page is organized top-to-bottom:

  1. Brand bar — Stride wordmark, project title, LearningMate co-mark.
  2. Top navigation — seven tabs: Course Inventory, Delivery Plan — May 21, Live Delivery Plan, Action Required, Live Status & RAG, Insights, and the new Drill Down.
  3. Filter bar (sticky) — refines the data from anywhere on the page.
  4. Delivery Window — 5 headline KPI tiles.
  5. By Month · Subject Breakdown, Daily Calendars, Deliveries by Month, Delivery Schedule — the body of the page.

Six fixed tabs (gray) + 30 state tabs. Users only edit cells inside state tabs; everything else is administrative.

The six fixed tabs

1. Course Master

The catalog of courses offered in this Subject × Grade. One row per (State, Course) pair. The key columns:

  • State — e.g., California, Texas
  • Base Course Number (BCN) — internal SKU like MA003FYCA01
  • Base Course Name — e.g., Math 3 CA
  • Focus Course Name — student-facing label, e.g., Math 3 Focus CA
  • Sem A/B — Full Year / SEM A / SEM B / SEM A+B
  • Derivative % — Master / 100% / 80% / 60% / 40%
  • Baseline Start Date — when Lesson Generation (stage 1) is supposed to start
  • Lessons Count — 90 (semester) or 180 (full year)

Course Master is the single source of truth. Change a state's start date here and every state tab and dashboard recalculates.

2. Config

Duration parameters for each of the 12 workflow stages — how many calendar days each stage gets between its baseline date and the next stage's baseline. Separate columns for Master courses vs. Derivative variants because a 40% derivative needs less time at certain stages. State-tab Baseline formulas read these durations to compute every per-lesson baseline date.

3. Dashboard

An aggregated read-only tab that rolls state-tab lesson data up into a single grid: each row = one (State, Course), columns = the 12 workflow stages. Useful when you want to scan one tracker's overall health.

4. Grade_Dashboard

A different rollup focused on 7 portfolio-level milestones: Completed in Brightspot, Added Ancillaries Links, BSP Course Validation, Publishing, Mastering, Green Lighting, Deliver and SO. One row per (State, Course); cols A–AP = identity + 7 milestones × 4 sub-cols (Baseline / Plan / Actual / Status) + 7 drift cols + course drift. This is the tab the HTML dashboard reads to build the portfolio-wide snapshot.

5. Dashboard_Export

A flat row-per-course table that flattens Grade_Dashboard + per-stage status counts (12 stages × 6 status categories = 72 cols + 23 core + 6 totals = 101 cols). Used by the Apps Script that builds the portfolio snapshot.

6. Dropdowns

Holds the data-validation lists used everywhere else (status values, SemA/B options, etc). Don't edit.

The 30 state tabs

Where the team actually does its work. One tab per MPS state. Each row in a state tab is one lesson; the 12 workflow stages cascade left-to-right (Lesson Generation → BrightSpot Signoff). The next two sections cover the per-cell layout and how to fill them in.

Protection: Course Master, Config, Dashboard, Grade_Dashboard, Dashboard_Export, and Dropdowns are fully admin-locked. On state tabs, identity columns (BCN, Focus Course, Sem, Baseline dates, formula columns, drift columns) are locked too. Users only edit the per-stage Assigned / Plan / Actual / Status cells, plus Lesson Name / Leverage / Drive links.

13 Workflow stages & statuses Lesson level (12 stages) + Course level (7 milestones)

Project 670 tracks delivery at two levels. At the lesson level, every lesson moves through a fixed 12-stage sequence on the state tab. At the course level, every course rolls up to 7 portfolio milestones on the Grade_Dashboard. Both levels share the same status vocabulary but differ in granularity.

Course-level workflow — the 7 portfolio milestones (Grade_Dashboard, cols G–AH)

Once a state's lessons are flowing through the 12 stages, the course as a whole still has seven downstream milestones before final delivery. These live on the Grade_Dashboard tab and are what the HTML Live Status / Insights / Drill Down (Grade Dashboard rollup) read from. Each milestone has the same 4 sub-columns (Baseline / Plan / Actual / Status) and the same 6-status dropdown vocabulary.

14 Example · Math Grade 3 · California A single-course state tab

Math Grade 3 California has one course: Math 3 Focus CA, Full Year, 180 lessons. The team's job is to fill in the per-stage Assigned / Plan / Actual / Status as work moves through the 12 stages.

Setup (done once)

  1. Open the Math Grade 3 tracker → click the California tab.
  2. Look at row 3 — the first lesson. Cols A–O are already populated: BCN MA003FYCA01, Focus Course Math 3 Focus CA, Sem Full Year, Lesson # = 1, Lesson Name = "Counting to 10".
  3. Col P (Baseline Start Date) shows e.g. May 14 — pulled from Course Master. This is Lesson 1's start date and seeds the cascade.
  4. Cols Q (Current Stage) and R (Lesson RAG) are formulas — leave them alone.

Editing Lesson 1 as work moves

Stage 1 lives in cols S–W:

Lesson 1, cols S–W (Stage 1: Lesson Generation)
STUVW
Headers row 1–2: "Lesson Generation (LDG)" merged across S:W
3 Team A May 14 May 14 YTS

Yellow = editable dropdown · Green = formula (locked) · Gray = empty / not yet filled

Day 1 (May 14): Set W (Stage 1 Status) to WIP - Outline. Plan and Actual stay as-is — no slippage yet.

Day 3 (May 16): Outline done. Set W to Completed - Ready for Lesson Development. Fill V (Actual) with May 16. Stage 2 starts — fill X (Stage 2 Assigned to) with the SME team.

What happens if Stage 1 slips: say content review takes longer and you know it won't finish until May 21. Update U (Plan) from May 14 to May 21. Three things happen automatically:

  • Col CD (Stage 1 Drift) flips from 0 to +7d.
  • Stage 2's Baseline (col Y) recomputes from Stage 1's Plan + Config duration. Every downstream baseline cascades.
  • Col R (Lesson RAG) may go from Green to Yellow or Red.

What the Drill Down shows you

After Math G3 California has some work logged, open the HTML dashboard → Drill Down tab → Math / Grade 3 / California → Load. Each Drill Down view exposes a different slice of what you edited:

  • Metadata: Start Date card = col P · Completed in Brightspot / Deliver and SO cards = Grade_Dashboard milestones 1 and 7 · Current Stage Summary = col Q aggregated across all 180 lessons.
  • Workflow Stages: top matrix shows, for Stage 1, how many of the 180 lessons are at each status. Accordion expands per stage and shows every lesson's S/T/U/V/W.
  • Lesson Cards: each lesson is one card; click to expand and see all 12 stages of that lesson at once.
  • Workflow Matrix: 180 rows × 12 colored cells. Find Lesson 1's row — orange (WIP) on day 1, green (Completed) by day 3.
  • Fixes & Blocked: any lesson + stage with Fixes Required or Blocked status surfaces here immediately.

15 Example · ELA Grade 3 · California

ELA Grade 3 California is structurally identical to Math G3 California — same 180 lessons, same 12 stages, same 5 cols per stage — but the work breakdown differs.

What's different from Math G3

  • Lesson Leverage (col L) matters more in ELA — many lessons reuse rubrics and exit-ticket frameworks across grades, so the column is filled in heavily.
  • Stage 4 (Media Development) is often marked NA for ELA reading lessons (no animation needed). Fill AL with NA and leave AJ/AK/AH blank — the downstream baseline still cascades from Stage 3's Plan.
  • Mindsmith Link (col N) usually points to a vocab-builder activity rather than an interactive simulation.

A typical week of edits

  1. Monday: Lessons 1–10 at Stage 1 WIP. Team A assigned, Plan dates on baseline.
  2. Wednesday: Lessons 1–5 finish Stage 1 — set W to Completed - Ready for Lesson Development, V to today's date. They flow into Stage 2.
  3. Thursday: Lesson 3 hits a content issue — set Lesson 3's W to Fixes Required - Content. Lesson 3 immediately appears in Drill Down → Fixes & Blocked with a "Stage 1: Lesson Generation" badge.
  4. Friday: SME team picks up Lessons 1, 2, 4, 5 at Stage 2 — set AB to WIP, fill X with assignees. Lesson 3 is still at Fixes Required.

17 How the data flows up From state tab → HTML dashboard

Once your team edits a state-tab cell, the change flows through three transformations before it appears on the HTML dashboard.

The three transformations

  1. State tab → Grade_Dashboard (in-spreadsheet). The Dashboard and Grade_Dashboard tabs inside each tracker are driven by formulas. As soon as a state-tab cell changes, the Grade_Dashboard row for that course updates within the same workbook.
  2. Grade_Dashboard → Portfolio snapshot (Apps Script, hourly). An Apps Script reads Grade_Dashboard A4:AP from each of the 28 trackers, plus per-stage status counts from Dashboard_Export. It writes Portfolio_Snapshot.json to a shared Drive folder.
  3. Portfolio snapshot → HTML dashboard (browser, on tab open). The Live Status / Live Delivery Plan / Insights / Drill Down tabs JSONP-fetch this file when you click them.

The Drill Down tab is special

Most live tabs read from the portfolio snapshot only. Drill Down does both:

  • The dropdowns and the Grade_Dashboard summary card come from the cached portfolio snapshot.
  • The full per-lesson data is fetched live on the Load click directly from the selected tracker's state tab. Each Load takes 2–3 seconds because it opens the spreadsheet in real time.

This means: edits show up in Drill Down on the very next Load click, while edits propagate to Live Status / Insights only after the next hourly snapshot run.

Status of the locks

The Apps Script Protect_Sheets.gs applies these locks portfolio-wide:

  • Course Master, Config, Dashboard, Grade_Dashboard, Dashboard_Export, Dropdowns → fully admin-locked.
  • State tabs → whole sheet protected, with editable ranges on cols D–H, K–O, and the per-stage Assigned (S/X/AC/…) / Plan / Actual / Status columns.
  • Baseline columns, Current Stage, Lesson RAG, summary and drift cols (CA–CO) → locked.
If a teammate edits a locked cell: Google Sheets pops a warning and the change doesn't save. This is intentional — it prevents accidental restructuring that would break the formulas and the portfolio snapshot.

18 Live Delivery Plan The same view as May 21, but always current

"Delivery Plan — May 21" is a frozen snapshot kept for reference. "Live Delivery Plan" is the same shape of report driven by the live portfolio snapshot.

What's identical to May 21

  • Same KPI tiles, calendars, schedule, export buttons.

What's different

  • Source data is live. Driven by the hourly portfolio snapshot.
  • Calendar has two modes. "Cleaner" / "Detailed" toggle changes how shifted-delivery dates are annotated.
  • Click any calendar cell with deliveries — a popup lists exactly which courses are scheduled for that day.
  • Status / Slip columns in the schedule reflect the M7 (Deliver and SO) milestone.
Effective date concept: the calendar groups deliveries by their effective date — Actual if delivered, otherwise Plan. Baseline is reference-only.

19 Action Required Lessons that should have started by now

Which lessons were supposed to begin work but haven't?

What "Action Required" means

A lesson surfaces when all three are true:

  1. Stage 1 Baseline (col T) is before today.
  2. Stage 1 Status (col W) is YTS, blank, or Not Started.
  3. The tracker is reachable when the overdue snapshot is built.

Why a separate timer

Scanning every lesson row in all 28 trackers takes ~5 minutes. So this tab has its own refresh:

  • Daily auto-refresh at 7 AM IST.
  • Manual refresh via the orange button (~5 min).

Grouping

Rows are grouped by course. Within each group: every overdue lesson with days-overdue count. Yellow = 1–3 days late, Red = 4+ days late.

How RAG is calculated Two layers: Lesson RAG and Course RAG

Project 670 computes RAG at two distinct layers. Both use the same color vocabulary (Red / Yellow / Green / Gray) but look at different inputs.

1. Lesson RAG — state tab, col R

One value per lesson row. Computed by formula across the lesson's 12 stages.

ColorWhen it triggers (any of these)
Red • Any of the 12 stages has Status = Blocked
• Any stage drift > 3 days (drift = Plan − Baseline)
• Lesson is overdue — col P (Baseline Start) is before today AND Stage 1 Status is still YTS / blank
• Any stage's Actual is > 3 days later than its Baseline (after Completion)
Yellow • Any stage has Status = Fixes Required
• Any stage drift is 1–3 days
Green • Every stage has drift ≤ 0 (on schedule or ahead) and no Blocked / Fixes Required
• Or every stage is Completed with Actual on or before Baseline
Gray • Col P (Baseline Start Date) is blank — the course doesn't have a committed start date yet, so RAG can't be computed

2. Course RAG — Grade_Dashboard, col F

One value per (State, Course) row. Aggregates worst-case signals across the 7 portfolio milestones and the lesson-level health rollup.

ColorWhen it triggers (any of these)
Red • Any of the 7 milestones has Status = Blocked
• Any milestone drift > 3 days (milestone Plan − milestone Baseline)
• >X% of the course's lessons are Red at the lesson level
• Course's Course Drift > 3 days
Yellow • Any milestone has Status = Fixes Required
• Any milestone drift is 1–3 days
• Some lessons are Yellow but none Red
Green • Every milestone on or ahead of Baseline, no Blocked / Fixes Required
• All lessons in the course are Green or Completed
Gray • Course's Baseline Start (Course Master) isn't committed yet
• Or Grade_Dashboard milestone 1 Baseline is blank — the course hasn't entered the development pipeline

Mental model: Lesson RAG is the “is this lesson on track” signal that lives next to each row on the state tab. Course RAG is the “is this course on track for delivery” signal that lives on Grade_Dashboard and powers the executive view. Lesson RAG feeds upward into Course RAG, but Course RAG also reads milestones (BSP Validation, Publishing, etc.) that don't exist at the lesson level.
Why this matters for editing: if a course shows Yellow on Live Status but you can't see why, drop into Drill Down for that state. Open the Workflow Stages view — any stage with a non-zero Fixes Required or drift count is a likely culprit. Or open Fixes & Blocked — if it shows any items, that's the Yellow trigger.

22 Drill Down One state, five live views

Other live tabs aggregate across the portfolio. Drill Down goes the other way: you pick one (Subject, Grade, State), and the dashboard pulls that state's full lesson grid live from Google Sheets and renders it in five complementary views.

How to use it

  1. Click the Drill Down tab (rightmost in the top nav).
  2. Pick Subject → Grade → State. Dropdowns cascade — only valid combinations are selectable.
  3. For Highschool, a Focus Course (HS) dropdown appears if the state holds multiple courses.
  4. Click Load. First load takes 2–3 seconds (Apps Script cold-start). Subsequent loads are faster.
  5. Use the view-toggle bar to switch between the five views.

A bordered table grouped by the value in col Q. Columns: Lessons / Red / Yellow / Green / Gray with colored headers. Lets you see at a glance where the 180 lessons are stuck and how many are Red.

Lesson Identity

Each row is a contiguous block of lessons sharing the same identity (BCN, Focus Course, Sem, Temp 1–5, Derivatives, Baseline Start). A full-year single-course state collapses 180 rows into one. Multi-course HS states show multiple blocks. "Lesson Range" reads "1 - 180".

Grade Dashboard rollup

Bottom of the Metadata view: a course-level table pulled from the live snapshot's Grade_Dashboard data. KPI tiles (Courses / Red / Yellow / Green / Drifted / Semesters), then one row per (Course, Sem) with all 7 portfolio milestones' statuses + drift.

View 2 · Workflow Stages (Q–BZ)

Top: 12 × 6 summary matrix

Twelve rows (one per stage) × six status columns (Done / WIP / Fixes / Blocked / YTS / N/A) + Total. Cells tinted by their status color — scan vertically for trouble spots.

Below: per-stage accordion

One card per stage. Header shows status pills ("44 done · 12 WIP · 1 fix · 5 YTS"). Click to expand and see every lesson in that stage with Lesson # / Lesson Name / Assigned to / Baseline / Plan / Actual / Status / Drift. Stage 1 expanded by default. Expand all / Collapse all buttons control the rest.

View 3 · Lesson Cards

One expandable accordion card per lesson. Header: Lesson # · Focus Course · Lesson Name · Current Stage · RAG pill. Click to expand and see all 12 stages stacked vertically. Useful when you want the complete vertical view of one lesson.
